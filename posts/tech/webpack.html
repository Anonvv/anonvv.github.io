Webpack 基础

08 / 10 / 2022

1. Webpack #

Webpack 是一个基于 Nodejs，遵循 Commonjs 规范的静态模块打包工具。

1.1. 核心 #

  1. loader：加载非 JS 模块；
  2. plugin：在 webpack 构建流程中注入扩展逻辑来改变构建结果。

1.2. 主要功能 #

  1. 代码转换
  2. 文件优化
  3. 代码分割
  4. 模块合并
  5. 自动刷新
  6. 代码校验
  7. 自动发布

2. 使用 #

2.1. 安装 #

# webpack-cli: 解析用户传入的参数   
$ npm i webpack webpack-cli

2.2. webpack 默认配置文件 #

  • webpack.config.js（常用）
  • webpack.file.js

2.3. 执行 webpack 打包 #

# npx: npm 5.2 之后的，默认执行 node_modules/.bin 下的文件
# 可以通过 --config 指定执行文件
$ npx webpack # 零配置打包
$ npx webpack --mode development # 开发模式
$ npx webpack --mode production # 生产模式

3. 插件 #

  • webpack-merge: 合并配置文件
  • webpack-dev-server: 开发环境服务器，在内存中打包，不会产生实体文件
  • html-webpack-plugin: 生成 html 模板文件，并自动插入 js
  • clean-webpack-plugin: 每次打包前清空目录
  • autoprefixer: 自动添加 css 前缀
  • mini-css-extract-plugin: 抽离 css 样式, 可以和 js 并行加载
  • optimize-css-assets-webpack-plugin：压缩 css 文件
  • terser-webpack-plugin：压缩 js 文件
  • babel：ES6+ 转化成 ES5
    • @babel/core：核心模块，转化代码
    • @babel/preset-env：所有把 ES6 转化成 ES5 的插件集合，解析 ES6 转化成 ES5
    • @babel-loader：默认调用 @babel/core 解析 js 文件
    • @babel/plugin-proposal-decorators：解析草案语法 - 装饰器
    • @babel/plugin-proposal-class-properties：解析草案语法 - class
    • @babel/plugin-transform-runtime"：节约代码

3.1. Loader（执行顺序：从下往上，从右往左） #

  • css-loader: 解析 css 语法，并把结果传给 style-loader
  • style-loader: 将解析的 css 变成 style 标签，插入到页面中
  • sass-loader: 调用 node-sass 处理 sass 文件
  • postcss-loader: 处理 css 前缀
  • file-loader: 处理图片模块
  • url-loader: 将小图片转成 base64

4. 优化 #

  1. 删除无用的 css 样式
    • purgecss-webpack-plugin: 只能配合 mini-css-extract-plugin 优化
    • glob: 查找匹配的文件
  2. 图片压缩
    • image-webpack-loader
  3. CDN 加载文件
    • add-asset-html-cdn-webpack-plugin
  4. 去除无用的代码
    • tree-shaking：默认只支持 ES6 语法（静态导入），只在生产环境下可用
    // package.json

 // 移除副作用代码，如果引用的变量没有使用，就删除

    "**/*.css" // css 文件不是副作用
